# 业绩总览卡片 FAQ 与常见误解

## 1. FAQ

### Q1. 累计收益和年化总收益为什么不一样

答：

- `累计收益` 讲的是报告区间内实际涨了多少
- `年化总收益` 讲的是把这段表现折算成一年后的收益率

所以它们本来就不是同一个概念。

### Q2. 年化主动收益和年化主动收益（几何）有什么区别

答：

- 两者都描述组合相对基准的超额收益
- 前者偏向常规年化超额收益口径
- 后者是几何复利口径

如果更关心长期复利意义上的跑赢，优先参考几何口径。

### Q3. 为什么过滤异常点后，收益走势图变了，但表格指标没变

答：

- 因为 `过滤异常点` 只对收益走势曲线生效
- 不会修改其他结果

这是产品设计，不是 bug。

### Q4. 为什么切换周频、月频后，指标也会变化

答：

- 因为频率切换不仅改变图上的采样点
- 也会影响部分收益和风险指标的观察频率

所以有变化通常是正常现象。

### Q5. 为什么有时会出现“包含转型前业绩”

答：

- 因为某些产品有转型、换壳或继承历史
- 系统允许选择是否把转型前业绩并入当前产品历史

勾选与否会真实影响结果。

### Q6. 为什么基准的胜率、盈亏比、相关性看起来很特殊

答：

- 因为这些值在基准侧不是按和组合完全对称的方式计算
- 当前兼容口径里：
  - 胜率固定为 `0`
  - 平均损益比为 `null`
  - `Spearman` 和 `Pearson` 固定为 `1`

### Q7. 为什么超额收益有算术和几何两条序列

答：

- `算术超额收益` 更直观
- `几何超额收益` 更接近复利意义上的真实超额表现

两者都合理，但不能直接混为同一口径。

### Q8. 为什么有些产品默认就是周频

答：

- 因为底层净值源或产品类型决定了默认频率
- 前端会根据 `originalNavFrequency` 和 `isWeek` 做初始化

### Q9. 为什么平均规模、持券集中度、年化转手率有时为空

答：

- 这些是附加持仓型指标
- 不是主收益风险链路的必需返回
- 往往需要触发特定 reload 或底层持仓数据完整时才会有值

### Q10. 如果只有曲线异常，应该先查哪里

答：

优先检查：

- 当前是 `收益走势` 还是 `净值走势`
- 是否勾选了 `过滤异常点`
- 当前频率
- 当前日期区间

### Q11. 如果只有指标异常，应该先查哪里

答：

优先检查：

- 是否设置了 `年化因子`
- 是否包含转型前业绩
- 当前频率
- 是否是组合侧异常还是基准侧异常

### Q12. `annualQ` 是不是改了底层净值

答：

- 不是
- 它主要影响部分年化换算指标的口径
- 不等于把原始净值历史改了

### Q13. 为什么收益走势和净值走势不是同一套序列

答：

- 因为这两块解决的是不同问题
- 收益走势看的是收益累计过程
- 净值走势看的是净值本身的历史轨迹

### Q14. 为什么基准有时没有完整图线或某些指标缺失

答：

常见原因有：

- 当前没有有效报告基准
- 基准净值历史不足
- 某些相对基准指标在基准自身没有业务意义

### Q15. 如果重构，只保留最终指标表可不可以

答：

- 不建议
- 这张卡片依赖的不只是表格指标，还包括收益序列、净值序列、频率切换和转型日期标记

只保留最终表格会丢掉很多前端真实契约。

## 2. 最容易造成误解的点

- 把 `异常点过滤` 误认为全量重算开关
- 把 `annualQ` 误认为净值清洗逻辑
- 把 `portfolio` 和 `benchmark` 当成完全镜像对象
- 把 `算术超额` 和 `几何超额` 混为一个概念
- 忽略 `包含转型前业绩` 对结果的真实影响
