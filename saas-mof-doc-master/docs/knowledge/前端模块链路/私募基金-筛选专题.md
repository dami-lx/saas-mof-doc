# 私募基金筛选专题

## 1. 这个专题关注什么

这篇文档只看私募基金筛选链路，不展开报告。

重点拆解：

- 普通筛选
- 高级筛选
- 动态字段
- 联想搜索
- 筛选方案

## 2. 前端页面入口

关键路由：

- `/app/fund/product/filter/private`

主要目录：

- `mof-web-fe/js/pages/product/filter/privateV2`

## 3. 前端 API 入口

关键文件：

- `mof-web-fe/js/pages/product/filter/privateV2/service.ts`

## 4. 前端字段模型

关键位置：

- `mof-web-fe/js/pages/product/filter/privateV2/columns/index.tsx`

当前可确认：

- 私募筛选字段同样沉淀在前端列配置中
- 大量字段定义了 `dynamicSearchKey`
- 私募字段口径明显不同于公募，不应强行复用

## 5. `mom` 后端入口

### 5.1 普通 / 高级筛选 / 动态搜索

主 controller：

- `mom/mom-web/src/main/java/com/datayes/web/mom/account/privatefund/controller/PrivateFundSearchController.java`

接口：

- `POST /api/account/pfund/ordinarySearch`
- `POST /api/account/pfund/advanceSearch`
- `POST /api/account/pfund/dynamicSearch`
- `POST /api/account/pfund/relatedSearch`
- `POST /api/account/pfund/searchByKey`

实现步骤：

1. `PrivateFundReqPreProcessor`
2. `PrivateFundAdvanceReqPreProcessor`
3. `PrivateAccountRankServiceImpl` / `PrivateAccountAdvanceRankServiceImpl`
4. `PrivateAccountListServiceImpl` / `PrivateAccountAdvanceListServiceImpl`
5. `PrivateResPostProcessor` / `PrivateDynamicResPostProcessor`

### 5.2 动态筛选字段

主 controller：

- `mom/mom-web/src/main/java/com/datayes/web/mom/account/privatefund/controller/PrivateFundDynamicFilterController.java`

接口：

- `GET /api/account/pfund/filter/field/dynamic`

已确认动态字段来源服务：

- `PrivateFundAwardService`
- `PrivateSecondaryStrategyService`
- `PublicSentimentService`
- `PrivateConScaleService`

### 5.3 联想搜索

主 controller：

- `mom/mom-web/src/main/java/com/datayes/web/mom/account/privatefund/controller/PrivateFundAssociateSearchController.java`

接口：

- `POST /api/account/pfund/associateSearch`

### 5.4 筛选方案

主 controller：

- `mom/mom-web/src/main/java/com/datayes/web/mom/account/privatefund/controller/PrivateFundQueryPlanController.java`

接口：

- `GET /api/account/pfund/plans`
- `POST /api/account/pfund/plans`
- `PUT /api/account/pfund/plans/{planId}`
- `DELETE /api/account/pfund/plans/{planId}`

实现语义：

- 存储为 `USER_PFUND_QUERY_PLAN` / `USER_PFUND_QUERY_SHOW_TEMPLATE`
- 属于用户配置体系的一部分

## 6. 数据来源与外部依赖

当前已确认：

- 前端打到 `rof_api`
- 后端当前仓库里有对等路径实现
- 动态字段服务有缓存设计
- 筛选方案持久化走用户配置模型

## 7. 公式与口径

私募筛选的核心不是公式，而是字段与搜索口径：

- 管理人奖项
- 基金公司奖项
- 管理人舆情
- 子策略
- 存续规模

这些字段明显体现出私募筛选有单独业务语义，未来要独立建模。

## 8. 重构时必须保留的契约

- 私募筛选字段模型不能被公募字段覆盖。
- `relatedSearch`、`searchByKey` 这类补充接口要保留。
- 私募筛选方案要与公募方案分开存储类型。
