# 模拟组合模块链路

## 1. 模块定位

这里的“模拟组合”模块主要覆盖：

- 情景分析 / FOF 组合
- 智能 FOF
- 相关的回测报告页

与“资产配置模块”的区别是：

- 资产配置更偏“创建可落地的配置型组合”
- 模拟组合更偏“情景、推荐、回测、虚拟 FOF、方案评估”

## 5. 下游依赖与数据来源

### 5.1 情景分析

当前已确认：

- 情景对象本身由 `ScenarioService` 管理
- 情景报告会继续下钻到净值 / 归因计算
- 前端通过 task 轮询等待异步任务完成

### 5.2 智能 FOF

当前已确认：

- `SmartFofRecommender`
  - 自定义推荐
- `SmartFofScenarioService`
  - 预设方案、推荐列表、回测结果
- `SmartFofSimulationService`
  - 仿真 / 回测
- `NavAttributionManager`
  - 回测结果中的净值归因填充

这说明模拟组合模块实际上横跨两种能力：

- 方案 / 场景管理
- 真正的回测与归因计算

## 6. 公式与计算口径

这个模块是明显的“计算型模块”，但公式主要不在前端。

已确认的公式入口包括：

- 情景分析报告：
  - `/api/attribution/simulation/v3/custom`
- 智能 FOF：
  - `SmartFofRecommender`
  - `SmartFofSimulationService`
  - `NavAttributionManager`
- `brain`：
  - 净值表现、模拟组合收益、归因、再平衡等端点

## 7. 重构时必须保留的契约

- 情景对象管理和回测计算要分层，不要把“场景 CRUD”和“回测引擎”耦合成一个类。
- 智能 FOF 的异步轮询协议需要保留，因为前端明显依赖 polling 任务模型。
- `/scenario/report/:id` 和 `/virtualFof/report/:id` 是结果展示层，不等于计算入口本身。
