# 公募基金报告专题

## 1. 这个专题关注什么

这篇文档只看公募基金报告链路，包括：

- 深度报告
- 简版报告
- 报告模板选择
- 报告页如何拿到账户上下文和缓存卡片

## 2. 前端页面入口

关键路由：

- `/report/:id(MUTUAL-.*)`
- `/simple-report/:id(MUTUAL-.*|PRIVATE-.*)`

路由定义：

- `mof-web-fe/js/v2/routes.ts`

主要容器：

- `mof-web-fe/js/v2/pages/IndependentReport/PageRouter/index.tsx`
- `mof-web-fe/js/v2/pages/simple-report`

## 3. 前端运行时结构

### 3.1 PageRouter 是报告分流器

关键文件：

- `mof-web-fe/js/v2/pages/IndependentReport/PageRouter/index.tsx`

当前已确认：

- `routeToNew` 会根据：
  - `accountInfo`
  - `template`
  - `location`
  决定走哪种页面实现
- `getReportType(location)` 决定返回：
  - `inner`
  - `public`
  - `private`
  - `virtualFof`
  - `default`

对公募基金报告：

- 会走 `public` 分支
- 页面组件实际渲染为 `PublicReport`

### 3.2 PageRouter 自带回退语义

`page-readme.md` 已明确写到：

- 加载中出错会回到老页面

这说明报告页当前是新旧实现共存状态，不是完全替换。

## 4. 前端 API 入口

关键文件：

- `mof-web-fe/js/v2/pages/IndependentReport/PageRouter/service.ts`
- `mof-web-fe/js/components/editTemplate/services.tsx`

已确认接口：

- `GET ${mof_api}/api/card/reportAccount/cached?accountId={accountId}&version=v1`
  - 获取报告页账户上下文、cacheId、cachedCardKeys
- `GET ${mof_api}/api/report/template/account/{accountId}?isBrief={bool}`
  - 获取账户默认模板或简版模板
- `GET ${mof_api}/api/sectionReport/reportTemplate?sectionReportId={sectionId}`
  - 截面数据模板

## 5. `mom` 后端入口

### 5.1 报告账户上下文

主 controller：

- `mom/mom-web/src/main/java/com/datayes/web/mom/card/CardViewCacheController.java`

关键接口：

- `GET /api/card/reportAccount/cached`

当前可确认的行为：

- 公募产品如果传入场内代码，会先转换成场外代码
- 记录报告访问埋点
- 调 `reportService.convertReportAccountForCached`
- 调 `paramService.getDefaultParam`
- 读取用户模板
- 生成 `cacheId` 和 `cachedCardKeys`
- 若无缓存则异步刷新报告缓存

这说明这个接口不只是“拿账户信息”，而是报告运行时装配入口。

### 5.2 模板获取

主 controller：

- `mom/mom-web/src/main/java/com/datayes/web/mom/card/ReportTemplateController.java`
- `mom/mom-saas/src/main/java/com/datayes/web/mom/card/section/SectionReportController.java`

关键接口：

- `GET /api/report/template/account/{accountId}`
- `GET /api/report/template/{templateId}`
- `GET /api/sectionReport/reportTemplate`

其中 `ReportTemplateController` 也对公募 accountId 做了场内/场外代码转换。

## 6. 下游依赖与数据来源

### 6.1 `mom` 卡片平台

负责：

- 模板解析
- cardKey 装饰
- 报告参数装配
- 缓存命中与刷新

### 6.2 `brain`

负责：

- 公募基金报告里的绩效、归因、风险、风格等分析计算

### 6.3 `pms` / 账户数据域

负责：

- 持仓
- 净值
- 账户基础信息

## 7. 公式与口径

公募报告的核心公式不在前端，而是分布在：

- 报告模板
- cardKey 对应卡片
- `CardDataController` / `CardViewController` 的 post 构造
- `brain` 端点

对未来重构最重要的不是直接复刻页面，而是保住这条链：

`accountId -> reportAccount/cached -> template -> cardKeys -> brain input -> rendered cards`

## 8. 重构时必须保留的契约

- 场内/场外公募代码转换逻辑不能丢。
- `cacheId`、`cachedCardKeys` 是报告性能与兼容层的一部分。
- 模板获取与页面路由分流必须保留，否则同一路由无法正确落到 `PublicReport`。
