# A股风格归因卡片客户业务说明

## 1. 文档目的

这份文档用于回答客户最常问的三个问题：

- 卡片里的收益、风险、暴露分别是怎么算出来的
- 这些数用了哪些数据
- 展示前做了哪些处理，尤其是“风格稳定性”蛛网图的月度口径

本文只讲业务口径，不展开接口、模块拆分和技术实现细节。

## 2. 先看这张卡片在展示什么

卡片名为 `A股风格归因`，分成两块：

- 上半部分三张柱图：`收益 / 风险 / 暴露`
- 下半部分三张蛛网图：`风格稳定性` 下的 `收益 / 风险 / 暴露`

卡片还有两个切换：

- `主动值`
  - 看组合相对基准的风格差异
- `绝对值`
  - 看组合自身的风格特征，不与基准做差

还有一个开关：

- `显示年化值`
  - 打开时展示年化口径
  - 关闭时展示非年化口径

前端帮助文案中对这张卡片的定义是：

- 使用全部持仓数据计算
- 收益、风险、暴露展示的是报告期累计数据

## 3. 主动值和绝对值分别是什么意思

这是客户最容易混淆的地方。

### 3.1 绝对值

绝对值反映“组合本身”在每个风格因子上的收益贡献、风险贡献和暴露。

可以理解为：

- 组合在大盘、成长、价值、动量、波动率等风格上，分别赚了多少收益
- 组合整体风险中，有多少来自这些风格
- 组合对这些风格到底偏多还是偏少

### 3.2 主动值

主动值反映“组合相对基准”的差异，也就是超配、低配、风格偏离带来的结果。

可以理解为：

- 相比基准，组合在哪些风格上多赚了收益或少赚了收益
- 相比基准，组合多承担了哪些风格风险
- 相比基准，组合在哪些风格上暴露更高或更低

业务上可以把它理解成：

- `绝对值 = 组合画像`
- `主动值 = 相对基准的偏离画像`

## 4. 顶部三张图怎么理解

上半部分的三张柱图展示的是报告期累计口径。

每一根柱子代表一个因子或一类因子。当前卡片里会把结果整理成三类：

- `行业`
  - 所有行业因子合并展示成一根“行业”柱
- `风格因子`
  - 如市值、价值、成长性、动量、波动率、流动性、杠杆率、盈利收益率、beta 等
- `特殊性成分`
  - 无法由行业和风格因子解释的剩余部分
  - 只会出现在收益和风险里，不会出现在暴露里

顶部标题上的“总和”不是另算的一套指标，而是当前图里所有展示项的直接加总。

## 5. 收益是怎么算的

### 5.1 单期收益的业务含义

单期收益的核心逻辑是：

- 先看组合在每个风格因子上的暴露有多大
- 再乘以该期这个风格因子的收益
- 得到这个因子在该期带来的收益贡献

### 绝对值收益

设：

- `w_p` = 组合权重
- `E` = 风格因子暴露矩阵
- `f` = 因子收益向量
- `r` = 个股收益向量

则：

- 组合在各风格上的暴露：`X_p = w_p^T E`
- 组合的因子收益贡献：`R_factor,p = X_p ⊙ f`
- 组合总股票收益：`R_p = w_p^T r`
- 组合特异收益：`R_specific,p = R_p - ΣR_factor,p`

这里的 `特异收益`，就是无法被已知行业和风格因子解释的剩余收益。

### 主动值收益

设基准权重为 `w_b`，则：

- 主动权重：`w_a = w_p - w_b`
- 主动暴露：`X_a = X_p - X_b = w_a^T E`
- 主动因子收益贡献：`R_factor,a = X_a ⊙ f`
- 基准总股票收益：`R_b = w_b^T r`
- 主动总收益：`R_a = R_p - R_b`
- 主动特异收益：`R_specific,a = R_a - ΣR_factor,a`

业务上可以理解为：

- 某个风格因子的主动收益为正，表示组合相对基准在这个风格上“做对了”
- 为负，表示这个风格偏离拖累了相对收益

### 5.2 报告期累计收益不是简单相加

顶部柱图展示的是报告期累计收益，不是把各期收益直接做普通求和。

系统会先算每一期的因子收益贡献，再做多期链接，使得：

- 各因子的累计贡献之和
- 能够回到整个报告期的组合收益或超额收益

因此更准确的表述是：

- 先做单期归因
- 再做多期累计链接
- 最后展示报告期累计收益贡献

这也是为什么顶部的收益图更适合回答“这段时间收益主要来自哪些风格”，而不是“某一个月的瞬时风格表现”。

## 6. 风险是怎么算的

### 6.1 单期风险的业务含义

风险不是看历史已经发生了多少亏损，而是看在风险模型下，组合风险有多少来自各个风格因子。

设：

- `Σ_f` = 因子协方差矩阵
- `s` = 个股特异风险
- `D = diag(s^2)` = 特异方差矩阵

### 绝对值风险

- 组合总风险：`σ_p = sqrt(Vol(w_p))`
- 组合风格方差贡献：`Var_factor,p = (X_p · Σ_f) ⊙ X_p`
- 组合风格风险贡献：`Risk_factor,p = Var_factor,p / σ_p`
- 组合特异风险贡献：`Risk_specific,p = (w_p ⊙ s^2)^T w_p / σ_p`

### 主动值风险

- 主动总风险：`σ_a = sqrt(Vol(w_a))`
- 主动风格方差贡献：`Var_factor,a = (X_a · Σ_f) ⊙ X_a`
- 主动风格风险贡献：`Risk_factor,a = Var_factor,a / σ_a`
- 主动特异风险贡献：`Risk_specific,a = (w_a ⊙ s^2)^T w_a / σ_a`

业务上可以理解为：

- 风险为正且数值大，说明该风格是当前风险来源的重要部分
- 风险很小，说明该风格对整体风险贡献有限

### 6.2 顶部风险图的报告期口径

顶部风险图虽然也叫“累计”，但它的业务含义和收益不同。

收益是做报告期累计链接；风险则是把报告期内各期的风险贡献汇总后，取报告期平均口径展示。

更直白地说：

- 收益回答的是“报告期累计赚了多少”
- 风险回答的是“报告期内平均来看，风险主要来自哪里”

默认展示为年化风险贡献。

如果关闭“显示年化值”，系统会按对应频率做去年化处理。

## 7. 暴露是怎么算的

暴露最容易理解，本质就是组合在各风格因子上的敏感度。

### 绝对值暴露

- `Exposure_p = X_p = w_p^T E`

### 主动值暴露

- `Exposure_a = X_a = X_p - X_b = w_a^T E`

业务上可以理解为：

- 暴露为正，表示更偏向该风格
- 暴露为负，表示更偏离或低配该风格
- 绝对值看组合本身的风格偏好
- 主动值看相对基准的风格偏离

顶部暴露图展示的是报告期平均暴露，而不是收益那种复合累计。

系统做法是：

- 先算每一期的风格暴露
- 再对报告期内各期暴露求平均

### 重要说明

暴露没有年化含义。

因此：

- 打开“显示年化值”时看到的暴露
- 和关闭后看到的暴露

数值实际上是相同的，只是界面统一沿用了同一个开关。

## 8. 这些数用了哪些数据

这张卡片的计算，核心会用到以下几类数据。

### 8.1 组合持仓数据

用于确定组合在每一期持有哪些A股，以及各股票权重。

业务含义：

- 这是风格归因的起点
- 没有持仓，就没有风格暴露和因子归因

### 8.2 基准成分和基准权重

用于计算主动值，也就是组合相对基准的风格偏离。

业务含义：

- 如果看的是主动值，就一定要有基准
- 没有基准，只能看绝对值，不能看相对偏离

### 8.3 个股收益

用于把持仓映射成组合收益、基准收益和特异收益。

业务含义：

- 收益归因最终还是要落到股票收益
- 风格因子只能解释一部分，剩下的就是特异收益

### 8.4 风险模型数据

这是风格归因最核心的一组底层数据，主要包括：

- 因子暴露矩阵
  - 每只股票在各风格因子上的暴露程度
- 因子收益
  - 每个风格因子在该期的收益表现
- 因子协方差矩阵
  - 用于计算风格风险
- 个股特异风险
  - 用于计算特殊性成分的风险

业务含义：

- 收益部分主要依赖“因子暴露 + 因子收益”
- 风险部分主要依赖“因子暴露 + 因子协方差 + 特异风险”
- 暴露部分主要依赖“因子暴露矩阵”

### 8.5 行业分类数据

行业数据用于把所有行业因子合并成一项“行业”展示。

因此客户在卡片里看到的是：

- 一根“行业”柱

而不是把申万一级、申万二级里的每个行业都逐一展开。

## 9. 这张卡片做了哪些展示处理

### 9.1 风格名称做了中文映射

底层风险模型中的因子代码，会在展示时映射成客户更容易理解的中文名称。

例如：

- `beta`
- `市值`
- `价值`
- `成长性`
- `动量`
- `波动率`
- `流动性`
- `杠杆率`
- `盈利收益率`

### 9.2 行业会被合并成“行业”

所有行业因子在展示层统一合并成一项 `行业`。

因此：

- 卡片不是在做“每个行业逐个展开”
- 而是在回答“行业这一大类因子整体贡献了多少收益、风险和暴露”

### 9.3 特异部分会被合并成“特殊性成分”

所有无法被行业和风格因子解释的剩余项，展示时统一归并为 `特殊性成分`。

这项会出现在：

- 收益
- 风险

不会出现在：

- 暴露

因为特异部分本身不是一个可解释的系统性风格因子，没有对应的风格暴露值。

### 9.4 数值展示格式

- 收益和风险按百分比展示
- 暴露按普通数值展示
- 小数保留两位

## 10. 风格稳定性蛛网图怎么理解

下半部分的“风格稳定性”不是报告期累计结果，而是把报告期内多期结果摊开来看，回答：

- 这个组合在不同月份里，风格收益是否稳定
- 风格风险来源是否稳定
- 风格暴露是否经常漂移

前端说明写得很明确：

- 使用 `t` 期持仓
- 计算 `t` 期暴露和 `t` 期风险
- 计算 `t+1` 期收益

业务理解可以写成：

- 风险和暴露看的是“当期持仓在当期风险模型下长什么样”
- 收益看的是“这期持仓带来了下一期什么结果”

## 11. 风格稳定性的月度口径

这是客户最关心、也最容易误解的点。

### 11.1 先算各期，再按自然月聚合

系统不是直接拿“月末一个点”来画蛛网图，而是：

1. 先算报告期内每一个归因期的收益、风险、暴露
2. 把日期截成 `YYYYMM`
3. 同一个自然月内如果有多个期点，先求月平均
4. 只保留最近 12 个月作为稳定性底表
5. 前端默认选最近 6 个日期展示

所以更准确的说法是：

- 蛛网图上的每个月，展示的是“该月内相关期点的平均值”

### 11.2 收益稳定性

收益稳定性的基础仍然是：

- 因子暴露 × 因子收益

但展示时不是报告期累计，而是“按月聚合后的多期分布”。

默认展示年化值。

如果切换到非年化，系统按频率做换算：

- `非年化收益展示值 = 年化收益展示值 / q × N`

其中：

- `q` = 年化系数
  - 例如月频通常对应 12，季频对应 4，半年频对应 2
- `N` = 报告期内参与计算的期数

因此，客户如果问“是不是先月平均再年化”，更准确的回答应是：

- 系统先算各期收益贡献
- 再按自然月做月平均
- 默认展示年化口径
- 切换非年化时，再按系统定义换算成非年化口径

它不是“先把月收益简单加总，再统一做一次年化”的粗略算法。

### 11.3 风险稳定性

风险稳定性优先使用专门的风格稳定性风险结果，而不是简单复用顶部累计图的中间结果。

它的业务含义是：

- 在每个观察月里，组合风险主要由哪些风格因子驱动

默认展示年化风险贡献。

切换到非年化时，系统换算为：

- `非年化风险展示值 = 年化风险展示值 / sqrt(q)`

### 11.4 暴露稳定性

暴露稳定性看的是：

- 组合在不同月份里对各风格因子的偏好是否稳定

系统也会先按自然月聚合，再展示最近月份。

但暴露没有年化含义，因此：

- `非年化暴露展示值 = 年化暴露展示值`

也就是说，暴露这张蛛网图的年化/非年化切换不会改变数值本身。

## 12. 给客户的简明结论

如果要用最简洁的话给客户介绍，可以直接这样说：

- 收益：看报告期内各风格给组合带来了多少收益贡献；主动值是相对基准的超额收益贡献，绝对值是组合自身收益贡献。
- 风险：看报告期内组合风险主要来自哪些风格因子；默认展示年化风险贡献。
- 暴露：看组合在各风格上的偏好和偏离程度；主动值是相对基准的暴露偏离，绝对值是组合自身暴露。
- 风格稳定性：不是看累计结果，而是把报告期内各期结果按自然月聚合后，观察收益、风险、暴露在不同月份是否稳定。
- 风格稳定性的月度蛛网图，严格说是“先按期计算、再按月平均、默认展示年化口径”；暴露不做年化变换。

## 13. 适合对外解释的标准话术

可以直接复用下面这段话：

> A股风格归因卡片本质上是在回答三个问题：这段时间组合主要靠哪些风格赚钱，主要承担了哪些风格风险，以及在风格上相对基准偏向哪里。  
> 收益是用组合在各风格因子上的暴露乘以因子收益得到的，并做报告期多期累计链接；风险是基于风险模型，把组合总风险拆分到各风格因子和特殊性成分上；暴露则直接反映组合在各风格上的敏感度。  
> 上方柱图看的是报告期整体结果，下方风格稳定性看的是分月后的稳定程度。稳定性图不是简单取月末点，而是先算各期数据，再按自然月求平均后展示；其中收益和风险默认显示年化值，暴露不做年化处理。
