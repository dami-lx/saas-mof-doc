# 私募基金报告专题

## 1. 这个专题关注什么

这篇文档只讨论私募基金报告链路，包括：

- 深度报告
- 简版报告
- 模板与账户上下文获取
- 报告页如何分流到新实现

## 2. 前端页面入口

关键路由：

- `/report/:id(PRIVATE-.*)`
- `/simple-report/:id(MUTUAL-.*|PRIVATE-.*)`

路由定义：

- `mof-web-fe/js/v2/routes.ts`

主要容器：

- `mof-web-fe/js/v2/pages/IndependentReport/PageRouter/index.tsx`

## 3. 前端运行时结构

### 3.1 私募报告路由分流

`PageRouter` 当前已明确存在 `private` 分支。

分流后：

- 仍渲染 `PublicReport`

这说明“公募报告页”和“私募报告页”目前在前端容器层共用实现，但输入上下文和模板不同。

### 3.2 上下文获取

关键文件：

- `mof-web-fe/js/v2/pages/IndependentReport/PageRouter/service.ts`

已确认接口：

- `GET ${mof_api}/api/card/reportAccount/cached`
- `GET ${mof_api}/api/report/template/account/{accountId}`

## 4. `mom` 后端入口

### 4.1 报告账户上下文

主 controller：

- `mom/mom-web/src/main/java/com/datayes/web/mom/card/CardViewCacheController.java`

关键接口：

- `GET /api/card/reportAccount/cached`

作用：

- 获取 `ReportAccount`
- 获取缓存卡片 key
- 构造默认报告参数
- 触发缓存预热或异步刷新

### 4.2 模板获取

主 controller：

- `mom/mom-web/src/main/java/com/datayes/web/mom/card/ReportTemplateController.java`

关键接口：

- `GET /api/report/template/account/{accountId}`

作用：

- 获取用户模板或默认模板
- 用 `cardTemplateManager` 做 cardKey 装饰

## 5. 下游依赖与数据来源

私募报告链路的下游结构与公募相似：

- `mom` 负责模板与参数编排
- `brain` 负责绩效、归因、风险等计算
- 底层账户与持仓数据来自账户/组合数据域

但需要强调的是：

- 私募报告和公募报告即使共用 `PublicReport`，模板和字段口径也未必相同

## 6. 公式与口径

私募报告同样应按以下顺序理解：

1. 账户上下文
2. 模板
3. 卡片集合
4. `brain` 计算结果

不要把 `PublicReport` 容器误认为公式所在处。

## 7. 重构时必须保留的契约

- `PRIVATE-*` 路由必须保持可独立识别。
- 私募报告模板要与公募模板保持可分流能力。
- 即使容器共用，也不能把公募/私募的模板、卡片、口径直接混并。
