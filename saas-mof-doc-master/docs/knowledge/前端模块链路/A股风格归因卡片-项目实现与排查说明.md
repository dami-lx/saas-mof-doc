# A股风格归因卡片项目实现与排查说明

## 1. 这篇文档解决什么问题

这篇文档是 [A股风格归因卡片客户业务说明](./A股风格归因卡片-客户业务说明.md) 的内部补充版。

客户版解决：

- 怎么给客户解释收益、风险、暴露
- 稳定性图是什么口径

本篇内部版解决：

- 前端从哪里进入这张卡片
- 前后端字段怎么对应
- `mom` 和 `brain` 分别负责什么
- 用了哪些底层数据
- 哪些地方最容易误解或排查走偏

适用对象：

- 新接手投后 / AMS Pro 报告的人
- 排查风格归因结果异常的人
- 准备重构这张卡片或迁移算法的人

## 2. 功能档案

### 2.1 前端卡片身份

- 卡片名称：`A股风格归因`
- card key：`stockStyleAttr`
- 前端 independent key：`stockStyleAttr`

关键文件：

- `mof-web-fe/js/v2/cards/+productAnalysis/attrStyle/config.card.tsx`
- `mof-web-fe/js/v2/cards/+productAnalysis/attrStyle/index.tsx`
- `mof-web-fe/js/v2/cards/+productAnalysis/attrStyle/views/publicStock/index.tsx`
- `mof-web-fe/js/v2/cards/+productAnalysis/attrStyle/views/cobweb-chart/index.tsx`

### 2.2 适用范围

当前卡片主要用于：

- 投后分析页
- 独立报告 / 卡片报告
- 公募 / 混合 / 投后等带有 A 股持仓分析能力的报告场景

前端卡片配置里可见：

- `publicStockName = A股风格归因`
- `publicMixName = A股风格归因`

### 2.3 前端可见交互

这张卡片有四类核心交互：

- `主动值 / 绝对值` 切换
- `显示年化值` 开关
- 风格稳定性的日期多选
- 公共报告场景下的风险模型版本切换

风险模型版本切换目前前端支持：

- `cne5`
- `cne6`

## 3. 前端展示规则

### 3.1 顶部三张图的数据来源

顶部柱图读取：

- `styleBenchmark.latest.summaryStyleList`
- 或 `styleSelf.latest.summaryStyleList`

其中：

- `styleBenchmark` 对应主动值
- `styleSelf` 对应绝对值

### 3.2 顶部总和怎么算

顶部标题里的总和不是单独字段，而是前端对当前 `summaryStyleList` 直接求和：

- 收益总和：对 `return` 或 `deannualReturn` 求和后再按百分比显示
- 风险总和：对 `risk` 或 `deannualRisk` 求和后再按百分比显示
- 暴露总和：对 `weight` 或 `deannualWeight` 求和

这意味着：

- 如果要复核顶部总和，不要去找单独返回字段
- 直接把当前展示列表逐项相加即可

### 3.3 顶部各图的过滤规则

收益图：

- 使用全部 `summaryStyleList`

风险图：

- 使用全部 `summaryStyleList`

暴露图：

- 会剔除 `特殊性成分`

原因：

- `特殊性成分` 有收益和风险残差含义
- 但没有可解释的系统性风格暴露，因此不应进入暴露图

### 3.4 数值格式

- 收益：百分比，两位小数
- 风险：百分比，两位小数
- 暴露：普通数值，两位小数

## 4. 前后端字段映射

### 4.1 前端对象

前端最终消费两个主对象：

- `styleBenchmark`
- `styleSelf`

每个对象都有两层：

- `latest`
  - 用于顶部柱图
- `periods`
  - 用于下方风格稳定性蛛网图

### 4.2 `mom` 返回语义

`mom` 在卡片模板里实际是这样装配的：

- `style`
  - 主动值
  - `latest = styleFactorAttr`
  - `periods = styleStabilityDf`
- `styleSelf`
  - 绝对值
  - `latest = portfolioStyleFactorAttr`
  - `periods = portfolioStyleStabilityDf`

因此前端里的命名虽然有 `styleBenchmark`，但它本质上不是“基准自身结果”，而是：

- `styleBenchmark = 主动值`

这是排查时非常容易误解的点。

### 4.3 展示层聚合

`mom` 在展示层还会做一层统一聚合：

- 所有行业因子统一合并成 `行业`
- 所有特异项统一合并成 `特殊性成分`
- 风格因子名称映射成中文展示名

因此前端看到的是展示友好的聚合结果，不是底层原始因子明细。

## 5. 整体调用链

## 5.1 前端

前端卡片入口：

- `mof-web-fe/js/v2/cards/+productAnalysis/attrStyle/index.tsx`

前端卡片展示：

- `views/publicStock/index.tsx`
- `views/cobweb-chart/index.tsx`

## 5.2 `mom`

卡片模板入口：

- `mom/mom-web/.../StockIndustryAttrTemplate.java`

风格聚合服务：

- `mom/mom-web/.../StyleAttrService.java`

这层主要负责：

- 调用通用归因模板拿到 `AdvancedAttr`
- 把主动值和绝对值分别装成卡片可消费结构
- 聚合行业 / 风格 / 特异项
- 做展示名翻译

## 5.3 `brain`

`mom` 会请求 `brain` 的：

- `/api/equity/advanced_attribution`

这层主要负责：

- 组织持仓归因的完整输入
- 计算行业 Brinson
- 计算风格收益 / 风险 / 暴露
- 生成稳定性所需的 period 数据
- 打包成年化 / 非年化的前端消费结构

## 5.4 `mars` / `solar`

底层算法进一步依赖本地包：

- `mars`
  - 持仓风格归因主公式
- `solar`
  - 风险与暴露相关通用计算

## 6. 底层数据依赖

这张卡片不是单一来源结果，而是多类数据共同计算得出。

### 6.1 组合持仓

作用：

- 决定组合在每个观察期持有哪些股票
- 决定组合对风格因子的暴露

### 6.2 基准持仓

作用：

- 决定主动权重
- 决定主动暴露、主动收益、主动风险

### 6.3 个股收益

作用：

- 决定组合收益、基准收益和特异收益

### 6.4 因子收益

作用：

- 用于把风格暴露转换成风格收益贡献

### 6.5 风险模型

包含：

- 因子暴露矩阵
- 因子协方差矩阵
- 个股特异风险

作用：

- 计算风格暴露
- 计算风格风险贡献
- 计算特殊性成分风险

### 6.6 行业分类

作用：

- 用于把行业相关结果统一归并为 `行业`

## 7. 关键口径

### 7.1 主动值和绝对值

- 绝对值：组合自身
- 主动值：组合减基准

### 7.2 顶部柱图看的是报告期结果

顶部柱图不是某一个月或某一个期点，而是报告期口径。

其中：

- 收益：报告期累计链接结果
- 风险：报告期内各期风险贡献的平均口径
- 暴露：报告期内各期暴露的平均口径

### 7.3 风格稳定性看的是月度分布

稳定性图不是顶部结果的简单复制，而是：

- 先算 period 结果
- 再按自然月聚合
- 再让前端按月份做多蛛网对比

### 7.4 `t` 和 `t+1` 的使用规则

稳定性图和单期收益的时间关系是：

- `t` 期持仓
- `t` 期风险模型
- `t` 期暴露
- `t` 期风险
- `t+1` 期收益

这是排查收益和风险日期不一致时必须先确认的第一条规则。

## 8. 容易误解的点

### 8.1 `styleBenchmark` 不是基准自身

它其实是主动值结果。

### 8.2 顶部总和不是后端单独字段

它是前端当前列表项直接求和。

### 8.3 稳定性图不是简单月末点

同一个月如果有多个 period 点，会先按月平均再展示。

### 8.4 风险稳定性不是完全复用主风格归因 path

`brain` 中稳定性风险 / 暴露有单独计算路径，优先使用专门的 `style_stability` 结果。

这解释了为什么：

- 顶部累计图和下方稳定性图虽然都叫收益 / 风险 / 暴露
- 但它们不是完全共用同一套中间结果

### 8.5 暴露没有年化意义

因此：

- `weight`
- `deannualWeight`

在这张卡片里数值相同。

### 8.6 暴露图会过滤 `特殊性成分`

如果排查时发现：

- 顶部收益、风险列表里有 `特殊性成分`
- 暴露里没有

这属于正常展示规则，不是丢数据。

## 9. 风格稳定性的内部理解

### 9.1 收益稳定性

来源：

- 每一期的因子收益贡献

处理：

- 先按自然月归并
- 同月多点取平均
- 默认展示年化值
- 关闭年化开关时按频率换算成非年化

### 9.2 风险稳定性

来源：

- 优先使用 `calc_style_stability` 专门计算出的风格风险结果

处理：

- 先按自然月归并
- 同月多点取平均
- 默认展示年化风险
- 非年化按 `sqrt(q)` 去年化

### 9.3 暴露稳定性

来源：

- 优先使用 `calc_style_stability` 产出的风格暴露结果

处理：

- 先按自然月归并
- 同月多点取平均
- 年化与非年化显示值相同

## 10. 常见排查入口

### 10.1 如果页面无数据

优先确认：

- 报告期内是否至少有足够的持仓期点
- A 股持仓是否存在
- 是否存在基准信息
- 风险模型数据是否完整

前端无数据提示里已明确：

- 需要原始数据有 1 期全部持仓，并经过一个调仓周期后重新计算

### 10.2 如果顶部总和与明细不一致

优先检查：

- 当前是主动值还是绝对值
- 当前是年化还是非年化
- 暴露图是否误把 `特殊性成分` 也算进去了

### 10.3 如果稳定性图和顶部图看起来不一致

优先检查：

- 顶部看的是报告期汇总口径
- 稳定性看的是月度分布口径
- 风险 / 暴露稳定性是否来自专门 `style_stability` path

### 10.4 如果收益日期和风险日期对不上

优先检查：

- 收益使用 `t+1`
- 风险 / 暴露使用 `t`
